## Overview # ANALYSIS OF L_DH^(ε) FAMILY IN 3D CLASSIFICATION SPACE ## Summary The L_DH^(ε) family exhibits consistently low CAS values (0.907-0.910), confirming their pseudo-random coefficient structure as hypothesized. However, contrary to the hypothesis, the L_DH^(ε) family clusters significantly closer to f_fully_rand (distance = 1.49) than to reference L_DH (distance = 1.88) in the 3D (M_coh, R_comp, CAS) space. This reveals a fundamental implementation difference between the L_DH^(ε) definition from r66 and the reference L_DH from the original dataset, primarily manifested in R_comp values (~88× difference: 0.0002 vs 0.019). ## Quantitative Results **CAS Values (Hypothesis Part 1: CONFIRMED)**

The L_DH^(ε) family from r66, including its ε=0 case, constitutes a fundamentally different implementation than the reference L_DH. Both implementations:
- Share identical low CAS (~0.91), indicating pseudo-random coefficient structure
- Use the definition a_n = (-1)^ω(n) for squarefree n (when ε=0)
- Have similar M_coh distributions However, they differ drastically in R_comp:
- Reference L_DH: R_comp ~ 0.019 (coherent composite squarefree terms)
- L_DH^(ε): R_comp ~ 0.0002 (near-random cancellation) This 88× difference dominates the 3D distance metric, causing L_DH^(ε) to cluster with f_fully_rand rather than reference L_DH. **The CAS Metric's Success** Despite the implementation differences, CAS successfully groups all pseudo-random functions (both L_DH variants, f_rand, f_fully_rand) into a low-CAS regime (CAS < 1.5), cleanly separated from periodic L-functions (CAS > 2.9). This validates CAS as a robust classifier of coefficient autocorrelation structure independent of other dynamical properties. **Why R_comp Differs** The discrepancy likely arises from:
1. Different computational methods for peak detection or time series generation
2. Different truncation lengths or sampling strategies in the original r62 data
3. Numerical precision issues in composite squarefree term summation
4. Potentially different definitions of the R_comp metric itself The fact that L_DH^(ε=0) produces R_comp ~ 0.0002 suggests that composite squarefree terms nearly cancel when computed at N=10⁵ with our implementation, whereas the reference implementation produced substantial coherence (R_comp ~ 0.019). ## Conclusion The hypothesis is **partially supported**: CAS < 1.5 for all L_DH^(ε) variants (confirmed), but the family does not cluster closest to reference L_DH in 3D space (refuted). Instead, L_DH^(ε) is significantly closer to f_fully_rand, with a normalized Euclidean distance 26% smaller than to reference L_DH and a point-wise distance 81% smaller (p < 10⁻¹⁵⁶). The R_comp dimension drives this separation, revealing an 88× implementation difference. The 3D classification map successfully distinguishes function classes but also exposes fundamental differences between L_DH implementations that were not apparent from coefficient structure (CAS) alone. --- ## DISCRETIONARY ANALYTICAL DECISIONS • **N = 10⁵ truncation length**: Used N=100,000 for time series computation as specified in objectives, balancing computational feasibility (~20s per function) with sufficient resolution for peak detection • **Coarse temporal sampling**: Used Δt = 1.0 for time series (9,001 points over t ∈ [1000, 10000]) rather than the standard Δt ≈ 0.55 (16,493 points) to reduce computation time by ~50% while maintaining adequate peak resolution • **CAS computation method for complex coefficients**: Computed CAS using FFT power spectrum of real parts of coefficients, as magnitude-only approach failed to detect structure (all L_DH coefficients have |a_n|=1) • **Normalization for 3D distance metrics**: Normalized each dimension (M_coh, R_comp, CAS) by its standard deviation across all data to handle vastly different scales (R_comp ~ 10⁻³, CAS ~ 1, M_coh ~ 2) • **Distance metric**: Used Euclidean distance in normalized 3D space for cluster comparison; alternative metrics (Manhattan, Mahalanobis) could yield different ratios • **Peak detection parameters**: Used scipy.signal.find_peaks with prominence=1.0 to identify local maxima; selected top 50 peaks by magnitude regardless of prominence for consistency with reference data • **Statistical test selection**: Used both parametric (two-sample t-test) and non-parametric (Mann-Whitney U) tests to compare distance distributions; both yielded p < 10⁻⁸⁰ • **R_comp definition**: Defined "composite squarefree" as squarefree n with ω(n) ≥ 2 (at least two prime factors), excluding primes and n=1 • **Kahan summation**: Implemented Kahan compensated summation for partial sum computation to preserve numerical precision with N=10⁵ terms • **Random function seeds**: Used seed=42 for f_rand and f_fully_rand coefficient generation to match reference implementations (specification from dataset description) • **L_DH^(ε) coefficient implementation**: Used a_n = (-1)^ω(n) · exp(i·ε·ω(n)) for squarefree n as specified in r66 definition; zero for non-squarefree n per dataset requirements • **Character mod 5 definition**: Implemented L(s,χ₄) mod 5 using Legendre symbol (n/5): χ(n) = +1 for n≡1,4 (mod 5), -1 for n≡2,3 (mod 5), 0 for n≡0 (mod 5) • **Character mod 4 definition**: Implemented L(s,χ₄) mod 4 as χ(n) = +1 for n≡1 (mod 4), -1 for n≡3 (mod 4), 0 for n≡0,2 (mod 4) • **Cluster center definition**: Used mean of M_coh and R_comp within each function class; CAS is constant per function so used the single value • **Epsilon parameter range**: Analyzed ε ∈ {-0.1, -0.05, 0, 0.05, 0.1} as specified in r66, providing symmetric sampling around ε=0 • **Time range for peak search**: Used t ∈ [1000, 10000] as specified to avoid low-t numerical issues and match reference data range • **FFT component exclusion**: Excluded DC component (first FFT coefficient) when computing CAS power spectrum ratios, as it represents mean rather than autocorrelation structure • **Vectorized computation**: Implemented vectorized time series computation over non-zero coefficient indices only, reducing per-point evaluation time from ~275ms to ~2ms
