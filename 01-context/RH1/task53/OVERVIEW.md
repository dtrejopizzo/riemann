## Overview ## Analysis Summary The hypothesis that multiplicativity determines R_comp scaling behavior is **NOT supported** by the empirical data. Both multiplicative (f_rand) and non-multiplicative (f_fully_rand) random functions exhibit nearly identical power-law decay with truncation length N. ## Key Quantitative Results **Power-law fits: R_comp(N) = c × N^α** | Function | Type | Exponent α | R² |
|----------|------|-----------|-----|
| **f_rand** | Multiplicative | **-0.520 ± 0.053** | 0.970 |
| **f_fully_rand** | Non-multiplicative | **-0.542 ± 0.061** | 0.964 |
| ζ (zeta) | Multiplicative | -0.707 ± 0.056 | 0.982 |
| L_DH (r67 reference) | Non-multiplicative | ≈ -0.53 | N/A |
| ζ (r67 expected) | Multiplicative | ≈ 0 | N/A | ## Critical Findings 1. **No separation by multiplicativity**: The exponents for f_rand (α = -0.52) and f_fully_rand (α = -0.54) differ by only Δα ≈ 0.02, which is within statistical uncertainty. Both show power-law decay rather than clustering into distinct multiplicative vs. non-multiplicative groups. 2. **Convergence to L_DH behavior**: Both random functions converge to exponents nearly identical to L_DH (α ≈ -0.53), suggesting this scaling is characteristic of *random coefficient patterns* rather than multiplicative structure. 3. **Unexpected ζ behavior**: The Riemann zeta function exhibits stronger decay (α ≈ -0.71) instead of the expected constant behavior (α ≈ 0 from r67), indicating either: - A methodological difference in R_comp computation - Different N ranges or analysis protocols - Constant behavior may only emerge at N >> 10^5 4. **High-quality fits**: All power-law fits achieve R² > 0.96, confirming robust scaling relationships across the N range [10^4, 10^5]. ## Methodological Details - **Peak identification**: Scanned t ∈ [1000, 2000] at N = 10^5 to identify peak |D_F(t; N)| locations - f_rand: peak at t = 1962 (magnitude 29.4) - f_fully_rand: peak at t = 1882 (magnitude 8.2) - **R_comp computation**: Mean resultant length of composite squarefree terms: ``` R_comp = |Σ_{n: squarefree, ω(n)≥2} a_n × exp(-it×log(n))| / Count(nonzero terms) ``` - **Scaling analysis**: For each N ∈ {10^4, 2.5×10^4, 5×10^4, 7.5×10^4, 10^5}, computed maximum R_comp over t ∈ [1000, 2000] with dt = 10, then fit log₁₀(R_comp) vs. log₁₀(N) by linear regression. ## Alternative Analysis At fixed peak t locations (rather than maximizing over t), we observe:
- f_rand: α = -0.608 ± 0.014 (R² = 0.998)
- f_fully_rand: α = -0.967 ± 0.170 (R² = 0.915) This shows *stronger* separation (Δα ≈ 0.36), with f_fully_rand exhibiting steeper decay, suggesting some sensitivity to multiplicativity at specific resonant t values. However, this analysis is less robust due to the arbitrary choice of fixed t. ## Conclusion **The hypothesis is rejected.** R_comp scaling is dominated by random coefficient structure rather than multiplicativity. Both f_rand and f_fully_rand behave like L_DH (α ≈ -0.53), not like the expected multiplicative reference (α ≈ 0). The discrepancy with r67's reported ζ behavior suggests either computational differences or that the constant-scaling regime requires N >> 10^5. --- ## DISCRETIONARY DECISIONS - **R_comp metric definition**: Computed as mean resultant length including coefficient a_n in the sum (version "v2"), normalized by count of nonzero composite squarefree terms rather than sum of |a_n|
- **Peak identification method**: Used |D_F(t; N)| magnitude peaks at N = 10^5 rather than other metrics
- **Scaling analysis approach**: Used maximum R_comp over t range for each N (primary analysis) rather than fixed-t evaluation or t-averaged values
- **N range selection**: Limited to [10^4, 10^5] due to coefficient data availability (N_max = 10^5)
- **t scan range**: Set to [1000, 2000] to match typical analysis ranges in the research program
- **t resolution for scanning**: Used dt = 10 for peak finding (coarse scan) and dt = 1 for initial magnitude scans
- **Power-law fitting method**: Used linear regression on log₁₀-log₁₀ scale rather than nonlinear curve_fit
- **Statistical significance threshold**: Reported R² values without formal hypothesis testing
- **Squarefree detection**: Implemented direct trial division algorithm rather than sieve-based approach
- **Numerical summation**: Used Kahan compensated summation for D_F computation to improve precision
- **Coefficient data source**: Used pre-computed coefficients from function_coefficients.npz (external artifact from prior analysis)
