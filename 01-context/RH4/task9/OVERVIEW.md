</results> <challenges>
- Critical caveat: `pc1_complex` and `H_complex` are *class-level* statistics (one value per class, replicated across all 200 peaks). The classification problem therefore effectively reduces to separating 8 training points (in Fold 1) or 10 training points (in Fold 2) in 2-D, and "generalization" to the held-out family means its single class-centroid lands on the correct side of a boundary defined by the training centroids. Bootstrap CIs over peak resampling are degenerate (all peaks in a class share identical features) and substantially under-state true uncertainty about the population of L-functions — they reflect only sampling noise within fixed, repeated coordinates.
- A more honest uncertainty estimate would require many independent realizations per class (different random subsets of n, alternative ω-stratifications, or simulated cross-term matrices), which the present derived feature table does not provide.
- All other concerns from the dataset description (S_7 leak at N=10⁶, etc.) are irrelevant here because S_k features were not used.

Geometrically, the cross-term matrix M_jk = E[Re(S_j conj(S_k))] compresses each class to a point that cleanly separates GRH-true functions (high pc1_complex, low spectral entropy H_complex) from violators (lower pc1_complex, higher H_complex), with the F4 family and F12 occupying the same violator quadrant. The next experiments should expose the classifier to violator constructions structurally distinct from Davenport–Heilbronn and the F12 chi3+chi5 combination.
