Direct repeated calls to pari.ramanujantau caused PARI stack overflow for bulk generation, while PARI's modular-form routine mfcoefs proved efficient and stable. (2) Coefficient generation: for F9 (Ramanujan Delta), I used PARI/cypari2 with mf=mfinit([1,12],1), F=mfeigenbasis(mf)[1], and v=mfcoefs(F, 1000000), then converted the returned PARI vector to a NumPy object array of exact Python integers and saved coeffs_F9_N1e6.npz. For F10 (the level-11 weight-2 newform 11.2.a.a), I used mf10=mfinit([11,2],1), F10=mfeigenbasis(mf10)[1], and v10=mfcoefs(F10, 1000000), converted to NumPy int64, and saved coeffs_F10_N1e6.npz. For F11 (Sym^2 Delta), I derived coefficients from F9 rather than calling a built-in symmetric-square routine: for each prime p<=10^6 I set b_p=tau(p)^2-p^11; for prime powers I used the Euler-factor recurrence c_{k+3}=b_p c_{k+2}-(p^11 b_p)c_{k+1}+p^33 c_k, which comes from the local factor (1-alpha_p^2 X)(1-alpha_p beta_p X)(1-beta_p^2 X); then I extended to all n<=10^6 by multiplicativity using a smallest-prime-factor sieve. I saved the result as coeffs_F11_N1e6.npz.
