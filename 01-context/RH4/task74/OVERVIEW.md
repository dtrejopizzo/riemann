</results> <challenges>
1. The aggregated Sk_complex_all_2200peaks.npz file was not present; per-class .npy/.npy_t files in the working directory contained the same content (200×8 complex S_k) and were used directly.
2. The (pc1, H) feature space is 2-dimensional with several centroids only ~0.05–0.3 z-units apart, which fundamentally limits 11-way bootstrap classification: feature noise from finite-n averaging is large compared to the inter-centroid spacing for the F1/F9/F10/F11 and F4/F5p/F5m clusters.
3. Macro OvR AUC required a probabilistic score; we used softmax over negative distances since the “simple SVM” suggested in the brief is degenerate when training on a single point per class. This score is fully ordinal-equivalent to centroid distance.
4. Multi-class AUC interpretation: a family-level metric was added because individual classes inside a tight cluster cannot be resolved at small n by construction — this is a structural property of the feature, not a sample-size artifact.
5. The class-level nature of the feature (caveat r34) means each of the 200 peaks within a class supplies essentially the same true (pc1, H) up to averaging noise, so the bootstrap quantifies estimator variance, not heterogeneity within a class.
</challenges> <discussion>
The spectral signature (pc1_complex, H_complex) is a clean class-level summary, but its discriminative power at small n is bounded by two factors: (i) the dimension of the feature is only 2, and (ii) several function classes share nearly identical centroids. Consequently, the AUC-vs-n curve rises smoothly but slowly with n; reaching AUC ≥ 0.95 in the full 11-class problem requires using ~75–90% of the available peaks per class, contradicting the practitioner-friendly hypothesis of n ≤ 20. When the task is reframed as identifying which family an unknown function belongs to (e.g., “standard-degree” L-cluster vs. “Davenport-Heilbronn-like” cluster vs. F12-type RH-violator), the picture is much more favorable: ~30 peaks suffice for AUC ≥ 0.95 and ~50 peaks give AUC > 0.97. This is the operationally meaningful regime: the spectral signature can robustly tell a Davenport-Heilbronn-type construction (F4/F5±) from a standard L-function (F1/F2/F9/F10/F11) with a few dozen peaks, but it cannot reliably tell, say, F1 from F9 without nearly the full set, because their (pc1, H) signatures coincide to within feature noise. Practically, the “cost” of identification depends sharply on what one wants to identify: family-level fingerprinting is cheap (~30 peaks), but full class-level identification is expensive (~150 peaks) and limited by feature collinearity. Adding more spectral dimensions (e.g., higher-order eigenvalue ratios or off-diagonal phase invariants) would be necessary to lift the F1/F9/F10/F11 and F4/F5p/F5m degeneracies and reduce n.
