<conclusion>
The Riemann zeta function ζ(s) exhibits non-monotonic evolution of its GEV shape parameter ξ in the vicinity of N=10⁵, with a local maximum at N=10⁵ (ξ = -0.006) followed by a dip at N=2×10⁵ (ξ = -0.062), confirming that the link between the M₄ structural anomaly and extreme value behavior is a general feature of arithmetic L-functions.
</conclusion> <methods>
For the Riemann zeta function ζ(s) with coefficients a_n = 1, I computed the Generalized Extreme Value (GEV) shape parameter ξ across N ∈ {10⁴, 5×10⁴, 10⁵, 2×10⁵, 5×10⁵, 10⁶}. Implementation:
1. Dirichlet polynomial evaluation: Implemented a Numba-accelerated function to compute D_ζ(t; N) = Σ_{n=1}^N 1/n^{1/2+it} using the formula n^{1/2+it} = n^{1/2} × exp(it×log(n)), with explicit calculation of cos and sin phases. 2. Grid evaluation: For each N, evaluated |D_ζ(t; N)| on a grid of 10,000 equally-spaced points in t ∈ [N, 2N], consistent with the established convention for N-dependence studies. 3. Block maxima computation: Divided each 10,000-point grid into 100 blocks of 100 consecutive points and extracted the maximum magnitude from each block, yielding 100 block maxima per N value. 4. GEV fitting: Applied scipy.stats.genextreme.fit() to the 100 block maxima for each N. The scipy implementation uses shape parameter c = -ξ, so I converted results to standard ξ notation. 5. Bootstrap confidence intervals: Computed 95% confidence intervals for ξ using bootstrap resampling with 500 resamples (np.random.seed(42)). For each bootstrap iteration, resampled the block maxima with replacement, fit the GEV distribution, and extracted the shape parameter. Confidence intervals were computed as the 2.5th and 97.5th percentiles of the bootstrap distribution. 6. Analysis: Examined the evolution pattern of ξ values across N, identifying local extrema and assessing monotonicity by counting increases and decreases in the sequence. Libraries used: NumPy 1.26.4, SciPy 1.16.2, Numba (JIT compilation), pandas, matplotlib, tqdm for progress tracking.
