# E73.238 - Curvature weight quotient for BSA-DIV

Date: 2026-07-14.

## 1. Purpose

E73.237 gives the proof-facing identity:

```text
F_L[B]=int_0^L K_L(t)(B^bal)''(t)dt,
```

where

```text
B^bal=B-B'(0)r_*,
F_L[r_*]=0.
```

This note rewrites the right-hand side as a finite coefficient sum.  The goal
is to expose the exact quotient in which the weights live.

## 2. Curvature weights

For each mesh frequency `omega`, define

```text
U_omega = int_0^L K_L(t)(-omega^2)e^(iomega t)dt,

Z_omega = int_0^L K_L(t)(2iomega-tomega^2)e^(iomega t)dt,

R_*     = int_0^L K_L(t)r_*''(t)dt.
```

Then E73.237 gives:

```text
F_L[B]
= sum_omega c_omega U_omega
 + sum_omega l_omega Z_omega
 - B'(0) R_*.                                      (1)
```

Using the derivative moment

```text
B'(0)=sum iomega c_omega+sum l_omega,
```

(1) becomes the pure coefficient sum

```text
F_L[B]
= sum_omega c_omega (U_omega-iomega R_*)
 + sum_omega l_omega (Z_omega-R_*).                (2)
```

Define the curvature quotient weights

```text
U^bal_omega=U_omega-iomega R_*,
Z^bal_omega=Z_omega-R_*.
```

## 3. Relation to the original weights

The same functional was already written as

```text
F_L[B]=sum c_omega W_omega+sum l_omega V_omega.
```

Therefore, on the admissible coefficient subspace

```text
sum c_omega=0,
sum l_omega=0,
B'(0)=sum iomega c_omega+sum l_omega,
```

the two weight systems are equivalent:

```text
sum c_omega W_omega+sum l_omega V_omega
=sum c_omega U^bal_omega+sum l_omega Z^bal_omega.  (3)
```

Equivalently,

```text
(W,V) - (U^bal,Z^bal)
```

belongs to the annihilator generated by the endpoint moments and the balanced
ramp relation.

This is the finite quotient statement behind SECOND-ABEL.

## 4. Current finite theorem

The remaining target is:

```text
CQ-DIV:
For eta=(I-P_w)xi_L and the finite coefficients c,l,

sum c_omega U^bal_omega
+sum l_omega Z^bal_omega
=O_M(L^-M).
```

This is equivalent to ETA-DIV, APR-U4, TRANS-CELL, and the scalar WRL gate, but
it displays the second-Abel curvature structure explicitly.

## 5. Why this matters

Generic bounds on `K_L` are known to be too weak.  The only possible gain is
orthogonality between `K_L` and the specific balanced curvature produced by
the Cauchy-Schur packet.

In coefficient language, that means the vector of curvature quotient weights
must be nearly orthogonal to the admissible coefficient vector.  The next
proof attempt should therefore target the finite bilinear pairing:

```text
< (c,l), (U^bal,Z^bal) >
```

using the extra algebraic structure of the coefficients, not absolute values
of `K_L` or generic packet norms.
