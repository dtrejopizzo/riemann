# E72.339 -- POSC as an exact Weil-Paley identity

**Purpose.** Replace the last contour target `POSC` by an exact compact-support explicit-formula
identity.  This removes the misleading derivative/smoothness language from E72.335--E72.338: the
paired contour is not controlled by absolute variation of `(B_z^M)'`.  It is the Weil functional of
the even compact packet `B_z^M(|t|)`.

## 1. The paired kernel is a bilateral Laplace transform

Let

```text
B(y)=B_z^M(y),        0<=y<=L,
B(L)=0,
F_B(w):=M_B(w)+M_B(-w)
      =int_0^L (e^(wy)+e^(-wy))B(y)dy.
```

Then the `POSC` integrand from E72.338 satisfies the exact identity

```text
(1/w)int_0^L (e^(-wy)-e^(wy))B'(y)dy
= F_B(w).                                            (1)
```

There is no endpoint residue: at `y=0` the factor `e^(-wy)-e^(wy)` vanishes, and at `y=L` the finite
packet satisfies `B(L)=0`.

Equivalently, define the even compact packet

```text
f_B(t):=B(|t|) 1_{|t|<=L}.
```

Then

```text
F_B(w)=int_{-L}^L e^(wt) f_B(t)dt.                   (2)
```

Thus `POSC` is a Paley-Wiener explicit-formula estimate for the test function `f_B`.

### Proof of (1)

Use

```text
(e^(-wy)-e^(wy))/w = -int_{-y}^y e^(wt)dt.
```

Then

```text
int_0^L B'(y)(e^(-wy)-e^(wy))/w dy
= -int_{-L}^L e^(wt) [int_{|t|}^L B'(y)dy]dt
= int_{-L}^L e^(wt)B(|t|)dt - B(L)int_{-L}^L e^(wt)dt.
```

Since `B(L)=0`, this gives (1)--(2).  If one instead integrates by parts directly, the boundary term
at `0` vanishes because the exponential difference is zero there.

## 2. Exact completed explicit formula for the packet

Let

```text
Z(w)=xi(1/2+w),
F_B(w)=int_{-L}^L e^(wt)f_B(t)dt.
```

For a zero-avoiding rectangle `R_T` in the `w`-plane symmetric under `w -> -w`,

```text
(1/2pi i)int_{partial R_T} F_B(w) Z'(w)/Z(w)dw
= sum_{w_rho in R_T} m_rho F_B(w_rho)
= 2 sum_{w_rho in R_T/+-} m_rho F_B(w_rho).           (3)
```

The last equality uses that `F_B` is even.  Thus the paired representative sum is one half of the
symmetric contour residue.

On the right vertical side `Re w=a>1/2`, writing `s=1/2+w`,

```text
Z'(w)/Z(w)
= 1/s + 1/(s-1) - (1/2)log pi
  + (1/2)psi(s/2) - sum_{n>=1} Lambda(n)n^(-s).       (4)
```

Because `f_B` is supported in `[-L,L]`, the prime part is finite after inverse Laplace:

```text
(1/2pi i)int_{Re w=a} F_B(w)
   [-sum_n Lambda(n)n^(-1/2-w)]dw
= -sum_{n<=e^L} Lambda(n)n^(-1/2) B(log n).          (5)
```

The polar and archimedean terms in (4) give the exact `WR`/archimedean functional already isolated in
E72.319--E72.330.  Since the left vertical side contributes the same amount by the parity
`F_B(-w)=F_B(w)` and `Z'(-w)/Z(-w)=-Z'(w)/Z(w)`, the representative paired zero sum is not a new
object:

```text
sum_{w_rho in Div(Z)/+-} m_rho F_B(w_rho)
= Arch_L(B) - sum_{n<=e^L} Lambda(n)n^(-1/2)B(log n), (6)
```

with the same finite-part convention as the Weyl-Riemann functional used earlier.

This is an identity, not an estimate.

## 3. The corrected analytic target

Let `W_T` be the finite zero window and define

```text
Rem_T^M(z)=sum_{w_rho notin W_T / +-} F_{B_z^M}(w_rho).
```

By (6),

```text
Rem_T^M(z)
= Arch_L(B_z^M)
 - sum_{n<=e^L} Lambda(n)n^(-1/2)B_z^M(log n)
 - sum_{w_rho in W_T / +-} F_{B_z^M}(w_rho).          (7)
```

Therefore the last open contour estimate is equivalent to:

```text
BER:
| Arch_L(B_z^M)
 - sum_{n<=e^L} Lambda(n)n^(-1/2)B_z^M(log n)
 - sum_{w_rho in W_T / +-} F_{B_z^M}(w_rho) |
<= C_{K,T}L^B.                                       (8)
```

`BER` is the balanced explicit-formula remainder.  It is the right analytic statement because:

1. it keeps the archimedean term and prime term coupled;
2. it never estimates the prime sum alone by absolute values;
3. the finite zero-window subtraction is explicit and fixed before the `L` limit;
4. no limiting divisor is inserted as an input.

## 4. Why this is not another numerical gate

The previous wrong turn was:

```text
Pair(w) -> |Pair(w)| -> ||(B_z^M)''||_1.
```

That loses the exponential cancellation in right shifted strips.

The correct route is:

```text
Pair(w) -> F_{B_z^M}(w)
        -> completed explicit formula
        -> coupled archimedean-prime finite functional.
```

So the next proof must act on the finite scalar functional (8), not on matrix norms.  In particular,
the estimate has to use the transformed Feshbach eigen-equation:

```text
(mu+e_pole)G_x(z)
= T_W02(z)[xi] - T_WR(z)[xi] - T_Prime(z)[xi],
```

because the three terms in (8) are precisely the three transformed pieces in this equation.

## 5. Immediate consequence

`POSC` is now reduced to a single analytic cancellation theorem:

```text
Balanced Feshbach-Weil theorem:
For the finite Feshbach packet B_z^M generated by the pole-even ground vector, the coupled functional
in (8) is polynomially bounded in every fixed shifted compact K and finite zero window W_T.
```

This theorem is narrower than the former `PACK-SMOOTH` target and avoids the incoherent ceiling.  It
is also the exact place where the proof must use the Weyl-Feshbach equation rather than standalone
prime estimates.
